  • 1. Self-Balancing Binary Search Tree: An AVL tree is a self-balancing Binary Search Tree where the difference between the heights of the left and right subtrees for any node is at most 1.
  • 2. Balance Factor: For any node, the balance factor is defined as: height(left_subtree) - height(right_subtree). * In an AVL tree, this factor must be -1, 0, or 1.
  • 3. Rotations for Balance: If an insertion (or deletion) causes the balance factor to exceed 1 or be less than -1, rotations (single or double) are performed to restore balance.
    • LL Rotation: Left-Left case, requires a single right rotation.
    • LR Rotation: Left-Right case, requires a left rotation on the child, then a right rotation on the parent.
    • RR Rotation: Right-Right case, requires a single left rotation.
    • RL Rotation: Right-Left case, requires a right rotation on the child, then a left rotation on the parent.
  • 4. Height Property: The height of an AVL tree with n nodes is O(\log n), ensuring efficient search, insertion, and deletion operations.
